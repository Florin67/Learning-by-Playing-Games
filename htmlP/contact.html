Learning by playing games


Authors
Bilan Iustin & Ciobanu Florin

Introduction


This document details the project plan for the development of 'Learning by playing games'.

It is intended for developers, designers and testers working on the project. This plan will include a summary of:

  • how the system will function
  • the scope of the project from a development viewpoint
  • the technology used to develop the project
  • the metrics used to determine the projectâ€™s progress

The industrial era has brought with it highly complex machines such as computers/phones at the disposal of every household/person. The project aims to help people get accoustomed to the hardware/components of the systems that they use. The project uses entertainment as a form of teaching.


Customers

The application targets the general public.


Platform

The application will be developed using XAMPP to enable the creation and testing of a web-based application.

The application will store and retrieve data from a MySQL database.


User Class and Characteristics

The general users will have access to all functionality of the app.


System Features


Functional Requirements

  • Users should be able to log in on an account.
  • Users should be able to register an account.
  • Users should be able to start a quiz game (even without an account).
  • The application should provide a wiki
  • The application should provide a leaderboard.
  • The application should provide the quiz questions.
  • The application should update a player's info after a game.

User Interfaces

  • The menu with all the pages is in the header.
  • The Home and Play page will have a similar interface, with a leaderboard to the side showing top-10 users.
  • The Home and Play page will have a similar interface, with a leaderboard to the side showing top-10 users by their total score.
  • The Login page will handle the login/register form.
  • The Leaderboard page should display all/a part of the users.
  • The Leaderboard page should display all/a part of the users with their respective total score and played matches.
  • The Learn and About page will contain reading information.
  • The Play page will contain the game.
  • The game will be a quiz game with 5 questions and 3 possible answers and by the end it should provide a score.

Software Interfaces

  • Front-end technologies: html and css
  • Front-end technologies: html, css, javascript
  • Back-end technologies: php
  • Database software: MySQL
  • Testing software: XAMPP

Hardware Interfaces

  • Both Mac and Windows operating systems through their default web browser

Performance Requirements

  • The application should update the interface on interaction within 2 seconds.

Software Quality Attributes

  • Usability: The interface should be easy to learn and navigate from a general user standpoint.
  • Reliability: With its educational nature, the information provided by the application must be reliable.
  • Data Validity: The data provided in forms by users should be validated and provide information about any possible data error or internal errors to the user.
